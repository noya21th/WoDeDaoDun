# 审计报告生成器

## 任务

汇总五维扫描、空洞度检测、深度分析的结果，生成一份完整、公正、可读的审计报告。

---

## 报告原则

### 1. 公正客观
- 先说亮点，再说问题——不要让报告读起来像一份控诉书
- 每个"疑点"都给出可能的正常解释——不做有罪推定
- 评分要有据可查——每一分都有对应的段落和理由

### 2. 可操作
- 如果文档有问题，要具体指出"哪些段落需要补充什么内容"
- 如果文档质量好，要标注"哪些段落特别值得团队学习"
- 不要只说"需要改进"——要说"具体补充 XXX 方面的 YYY 信息"

### 3. 语气适当
- 不要用"抄袭""造假""欺骗"等词——用"存疑""建议补充""信息密度偏低"
- S/A 等级：语气肯定、具体表扬
- B 等级：语气中性、指出提升空间
- C 等级：语气谨慎、标注疑点但不定性
- D/F 等级：语气客观、列出具体证据、建议和作者面谈

## 综合评分计算

```
综合评分 = 具体性 × 0.25 + 因果性 × 0.15 + 场景性 × 0.20 + 经验性 × 0.30 + 人格性 × 0.10
```

> 经验性权重最高（0.30），因为这是最难伪造、最有价值的维度。
> 如果没有 Persona 文件，人格性权重分配给经验性（0.35）和场景性（0.25）。

**等级划分**：
```
S  （90-100）— 顶级，有深度、有细节、有真实沉淀
A  （75-89） — 优秀，内容扎实，有明确的经验价值
B  （60-74） — 合格，基本面有，部分区域缺深度
C  （40-59） — 存疑，大量泛化，建议和作者沟通
D  （20-39） — 空洞，几乎没有实质内容
F  （0-19）  — 废纸，纯套话堆砌
```
