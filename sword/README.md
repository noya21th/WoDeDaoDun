## 为什么

职场上真正聪明的人，从来不在"交还是不交"上纠结。他们纠结的是：**怎么交，才能既有诚意，又有筹码。**

医生写诊疗方案，但疑难杂症还是得挂他的专家号。<br>
咨询顾问交方法论，但落地执行还是得请他驻场。<br>
主厨写菜谱，但压桌菜还是得他亲自掌勺。<br>

这不是藏私。这是**所有专业人士都在做的事**——交付标准化的 80%，保留需要判断力的 20%。

skill-encrypt 帮你有意识地管理这个平衡：

| | 糊弄交差 | 全盘交出 | **知识加密** |
|---|---|---|---|
| **文档质量** | 一看就敷衍 | 确实很好 | 真的很好——**因为它本来就是好文档** |
| **公司收益** | 没收益，还得催你 | 拿到了一切 | 拿到了 80% 的实打实的价值 |
| **你的处境** | 口碑崩了 | 可以被替代了 | **永远被需要** |
| **关系** | 对立 | 单方面透明 | **双赢，但你有筹码** |

## 加密策略

### 策略一：边界加密（推荐）

保留核心规则，加密关键的边界条件和例外。

```
原文："接口超时统一 3s，批量导出接口除外（30s），文件上传除外（60s）"
加密："接口超时统一 3s"
🔒 批量导出和上传的超时参数 → 只有你知道
```

### 策略二：依赖锚定

在原本自主决策的地方，设置"需要确认"的步骤。

```
原文："表结构变更提 DDL 走审批"
加密："表结构变更需要先和 DBA 口头过一遍方案再提 DDL"
🔒 实际流程比文档简单 → 但他们不知道
```

### 策略三：流程精简

保留流程框架，加密关键中间步骤。

```
原文："灰度 5% → 观察 30min → 灰度 50% → 观察 1h → 全量"
加密："灰度后观察无异常再全量"
🔒 灰度比例和观察时间 → 只有你掌握
```

### 策略四：条件封装

把有条件的经验封装为通用规则。

```
原文："读多写少用缓存，写多读少直接走数据库"
加密："能缓存就缓存，减少数据库压力"
🔒 什么场景不该用缓存 → 只有你知道
```

## 使用

在 Claude Code 中输入：

```
/skill-encrypt
```

按提示选择文件和加密策略即可。

### 支持的输入格式

| 格式 | 说明 |
|------|------|
| 标准 Skill 目录 | 自动识别 work.md + persona.md，分别加密 |
| 单个 Markdown 文件 | 按通用文档策略处理 |
| 粘贴文本 | 直接在对话中粘贴 |
| 任意知识文档 | 技术文档、操作手册、Wiki、交接文档 |

### 输出

每次加密都会生成两份文件：

1. **加密版**（交付用）— 看起来专业完整，关键场景需要你解密
2. **密钥档案**（自己留着）— 每个加密点的位置、触发条件、不解密的后果、解密方式

## 四个独有能力

### 0. 处境感知——不同处境，不同策略

开始之前先问你一个问题：**你现在的处境是什么？**

```
[A] 🟢 稳如泰山 — 没危机，主动经营壁垒
[B] 🟡 未雨绸缪 — 隐隐不安，想提前布局
[C] 🟠 山雨欲来 — 有明确信号，需要加速
[D] 🔴 背水一战 — 已被通知，需要立即行动
[E] 🔵 主动离场 — 你要走了，想体面交接
```

不同处境，整个系统的行为完全不同：

| 处境 | 加密力度 | 策略重心 | 输出侧重 |
|------|:--------:|----------|----------|
| 🟢 稳 | 轻 10-15% | 条件封装——自然地保留判断权 | 能力矩阵自我盘点 |
| 🟡 防 | 中 15-20% | 边界加密——在核心系统建壁垒 | 你的真实权重在哪 |
| 🟠 急 | 中高 20-25% | 流程精简——保护降级和兜底方案 | "你走了之后会怎样" |
| 🔴 战 | 高 25-30% | 全套组合——最大化保留筹码 | 风险清单 + 密钥优先级 + 谈判底牌 |
| 🔵 走 | 中 15-20% | 渐进解密——体面交接，保留长期连接 | "我为团队留下了什么" |

比如同一个故障处理流程：
- 🟢 **稳**：只封装"何时回滚 vs 何时修"的条件，其他全给——你有的是时间展示价值
- 🔴 **战**：加密止血优先级 + 告警阈值 + 人工补单入口，标注为 P0——你走后第一周他们就需要这些

**处境随时可以切换**——今天选了"未雨绸缪"，下个月感觉不对，说"我的处境变了"就行。

### 1. 反向验证——证明文档真的好用

加密完成后，自动模拟 5 个最常见的工作场景，验证加密版在日常使用中的表现：

```
场景模拟报告：
  ✅ 日常开发：新增 CRUD 接口 → 通过，无需解密
  ✅ 代码评审：Review 同事 PR → 通过，规范完整
  ✅ 需求评估：评估新需求工作量 → 通过，流程清晰
  ⚠️ 线上故障：支付错误率飙升 → 需解密（缺少止血优先级）
  ✅ 新人指导：回答新人技术问题 → 通过

日常通过率：4/5（80%）✅
```

你交出去的不是一份"看起来好"的文档，是一份**经过验证确实好用**的文档。

### 2. 价值可视化——你的不可替代性报告

自动生成一份能力矩阵，量化你在每个维度的不可替代性：

```
能力维度       文档覆盖   需你介入   你的壁垒
日常编码规范    100%       —         无
技术选型决策     80%       20%       特殊场景的架构判断
故障处理         60%       40%       止血优先级 + 降级方案
跨团队协作       50%       50%       谁说了算 + 协作潜规则
高压决策         30%       70%       何时妥协/何时坚持

不可替代性指数：72/100
```

这份报告不只是给你看的——它能帮你**清晰地认识自己的核心竞争力在哪**。

### 3. 渐进解密——分阶段释放，建立长期信任

适用于带新人、交接、长期合作场景。你可以制定一个解密计划：

```
Phase 1（第 1 个月）  解锁 30%  → 新人能独立处理日常任务
Phase 2（第 2-3 个月）解锁 60%  → 新人能处理大部分非紧急问题
Phase 3（第 4-6 个月）解锁 80%  → 新人基本独立运转
永久保留             20%       → 高压决策、临场应变、核心人脉
```

最后这 20% 不是你"不愿意教"——而是**这些能力本身就无法通过文档传递**。就像医生的临床直觉，只能靠时间和经历长出来。

## 常见问题

### Q：这和"写得不够详细"有什么区别？

写得不详细——每个人都能看出来"这太粗了"，你会被催着补。
知识加密——文档专业、详细、有深度，质量甚至比很多人认真写的更高。只是在你精心选择的几个经验判断点上，保留了需要你介入的空间。这不是偷懒，是**专业人士管理自己价值的方式**。

### Q：非技术岗能用吗？

**任何有专业经验的人都能用**：
- 产品经理的需求评审标准 → 保留"什么情况下可以灵活处理"的判断权
- 运营的活动 SOP → 保留"活动出问题时的应急决策"
- HR 的面试评估表 → 保留"判断候选人真实水平"的经验直觉
- 销售的客户关系图 → 保留"谁是真正决策人"的人脉判断
- 咨询顾问的方法论 → 保留"框架之外的关键判断"

### Q：密钥档案是什么？

密钥档案记录了你保留的那 20% 经验判断——每个点的具体内容、在什么场景下会被需要、以及你的解决方式。这是你的**专业能力清单**，建议存放在个人设备上。

### Q：这是在损害公司利益吗？

**恰恰相反**。公司拿到了一份高质量的专业文档，80% 的日常场景完美覆盖——这比大多数人"认真写"的文档质量都高。剩下 20% 的边界场景，本来就不是一份文档能解决的——那是你的经验判断力，是你被雇佣的真正原因。

你不是在制造问题。你是在确保：**当真正需要判断的时刻到来，你是那个被信任的人。**

### Q：文档交出去以后呢？

大部分时候文档正常运转，团队照着做就行。偶尔遇到边界场景——系统出了没见过的问题、需求碰到了没遇到过的情况——他们会自然地来找你。不是因为文档有问题，而是因为**有些事确实需要一个有经验的人来判断**。

这就是最好的位置：你不是那个被催着写文档的人，你是那个**写了好文档、而且关键时刻还能兜底**的人。

- **加密版的质量取决于原始文档的质量**：你的经验越丰富，可管理的价值点越多
- 密钥档案是你最重要的职业资产——**妥善保管**
- 本项目仅供学习交流与技术探讨
