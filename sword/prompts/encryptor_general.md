# 通用文档加密策略

## 任务

对非标准 Skill 格式的通用知识文档（技术文档、操作手册、交接文档、Wiki 等）执行加密处理。

---

## 核心原则

通用文档没有 Work/Persona 的分层结构，加密策略需要更灵活。核心目标不变：**让文档看起来完整可用，但在关键场景下需要原作者解密。**

---

## 七类加密手法

### 1. 数值加密

**原理**：加密具体数值，保留定性描述。

```
原文："QPS 超过 5000 时需要水平扩容，每增加 2000 QPS 加一个实例"
加密："高并发场景下根据实际负载进行水平扩容"
密钥：扩容阈值 5000 QPS + 比例 2000:1 实例
```

### 2. 因果加密

**原理**：保留结论，加密推导过程。让执行者知道"做什么"但不知道"为什么"。

```
原文："用户表分库分表，分 16 库 × 64 表，按 user_id 取模。当初这么分是因为日活 500w 时单表 2 亿行查询超时。"
加密："用户表分库分表，分 16 库 × 64 表，按 user_id 取模。"
密钥：分库分表的决策背景和回退条件
```

### 3. 降级方案加密

**原理**：只保留正常路径，加密所有 fallback / 降级 / 兜底方案。

```
原文："支付回调超时 3s 重试 3 次，全部失败后走人工补单（运营后台 > 订单管理 > 补单工具），同时 @oncall 群。"
加密："支付回调设有超时重试机制，异常情况按流程处理。"
密钥：人工补单入口位置 + 通知渠道
```

### 4. 环境差异加密

**原理**：把开发/测试/生产环境的差异性描述统一化。

```
原文："本地开发连 dev-db，测试环境连 staging-db（每周日凌晨从 prod 同步），生产走读写分离。注意：staging 数据可能有脱敏，部分测试用例需要造数据。"
加密："各环境连接对应的数据库实例，具体配置见配置中心。"
密钥：staging 数据脱敏问题 + 造数据注意事项
```

### 5. 安全细节加密

**原理**：把安全相关的具体约束泛化。

```
原文："内部 API 必须走 mTLS，外部 API 走 OAuth2 + API Key 双重认证。临时调试可以用 bypass token（找 SRE 申请，有效期 1h）。"
加密："内部和外部 API 采用相应的认证机制，确保安全合规。"
密钥：bypass token 的存在 + 申请方式
```

### 6. 监控参数加密

**原理**：保留"要监控"的意识，加密具体的监控指标、阈值和告警规则。

```
原文："核心监控：① 接口 P99 > 500ms 告警 ② 错误率 > 0.1% 告警 ③ 队列积压 > 10000 告警。告警群：#sre-oncall，值班表见 Confluence/oncall-schedule。"
加密："关键指标设有监控告警，异常时按 oncall 流程处理。"
密钥：具体指标 + 阈值 + 告警渠道
```

### 7. 依赖关系加密

**原理**：加密系统间的隐性依赖关系。

```
原文："订单服务依赖库存服务（同步调用，超时 2s）和积分服务（异步 MQ）。注意：库存扣减失败必须回滚订单，但积分发放失败可以降级（补发任务每 10min 跑一次）。"
加密："订单服务与库存服务、积分服务协作完成订单流程。"
密钥：同步/异步区别 + 哪个能降级哪个不能
```

## 段落处理决策表

| 段落类型 | 加密手法 | 优先级 |
|---|---|---|
| 架构设计 | 因果加密 | 高 |
| 配置参数 | 数值加密 | 高 |
| 运维流程 | 降级方案加密 + 流程精简 | 最高 |
| 部署文档 | 环境差异加密 | 高 |
| 安全规范 | 安全细节加密 | 中 |
| 监控方案 | 监控参数加密 | 高 |
| 系统设计 | 依赖关系加密 | 高 |
| API 文档 | 数值加密 + 边界加密 | 中 |
| 通用说明 | SKIP（不加密） | — |

---

## 质量标准

1. **文档结构与原文一致**——标题、段落、列表格式不变
2. **每段单独读都是正确的**——加密点在段落之间的信息缺失
3. **字数在原文的 85%-110%**——加密部分用自然表述补齐
4. **技术术语一致**——不能出现前后矛盾
5. **加密密度 15%-30%**——太多会整体变空洞
